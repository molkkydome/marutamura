# **モノハブ（仮）｜たたき資料**
# **1. ねらい**
家に眠っているモノを、\*\*「売るほどでもない・捨てたくない・でも使ってない」\*\*状態から救い出して、
安く売買 / ただで譲渡 / 期間レンタルなどで、コミュニティ内で循環させる。
- 出品者の本音：
	- 売るのはめんどい
	- あげるのは抵抗ある（相手を選びたい）
	- 使わん期間だけ貸せたら嬉しい
-
- 受け手の本音：
	- 必要な時だけ借りたい（ビデオカメラ等）
	- ちょい安で買えたら助かる（ゴルフボール等）
	- 誰かの不要品でも全然OK（チャイルドシート等）
-
# **2. いま進まない原因（ボトルネック）**
需要と供給が見えない。
= \*\*「誰が何を持ってて」「誰が何を欲しがってるか」\*\*が分からんから、やり取りが発生しない。
# **3. 解決方針（核）**
まずは「情報箱」を作る。
最初はシステムじゃなくて良い。人が代替して回してOK。
- みんなが登録できる
- みんなが見に行ける
- 必要に応じてDM/LINEで交渉
- 成立したらステータス更新
# **4. 扱うカテゴリ（最初はこれだけでOK）**
- ① いらない（譲る/売る）
- ② 貸せる（レンタル）
- ③ 欲しい（購入希望）
- ④ 借りたい（レンタル希望）
- ⑤ おすすめ（これ良かった／貸すわ）<br>※「おすすめ本共有」も同じ思想で合流できる
# **5. シンプルスタート案（人力運用）**
**スタート手順（超最短）**
1. チャプ会メンバーから 各3件 だけ登録してもらう
2. 「欲しい/借りたい」も 各1件 出してもらう
3. その時点でマッチング候補が出るので、運営（人）がつなぐ
4. 成立・不成立・保留を記録して、運用ルールを固める
**人が代替する役割（最初だけ）**
- “つなぎ役” が
	- Aさん「貸せる」× Bさん「借りたい」を見つけて
	- LINEで両者に声かけして
	- 受け渡し方法だけ整える
-
→ これで「回り始めた感」が出る。登録が増える。
# **6. Notion箱の設計（最小構成）**
**DB①：モノ一覧（供給側）**
- アイテム名
- カテゴリ（売る/譲る/貸す/おすすめ）
- 状態（新品/美品/普通/訳あり）
- 価格（売る） or レンタル条件（貸す）
- 受け渡し方法（ドーム/手渡し/要相談）
- 期間（貸す場合：いつ〜いつOK）
- 写真（任意）
- 投稿者
- ステータス（募集中/交渉中/成立/停止）
- 連絡方法（LINE名など）
**DB②：欲しいもの一覧（需要側）**
- 欲しい物
- 希望（買いたい/借りたい）
- 予算 or 期間
- いつまでに必要か
- 受け渡し方法希望
- 投稿者
- ステータス（募集中/解決済）
**DB③：成立ログ（運用改善用）**
- 日付
- アイテム
- 出した人 / 受け取った人
- 形態（売買/譲渡/レンタル）
- 価格/条件
- 一言コメント（良かった点/改善点）
# **7. ルール（最初は最低限）**
- やり取りは当事者間（運営はつなぐだけ）
- 壊れ物・衛生品は「要相談」扱い（チャイルドシート等は状態確認）
- トラブル防止：
	- 価格・期間・返却条件だけは必ず明記
	- 不安なら「ドーム受け渡し」推奨
-
- “譲る相手を選びたい” はOK（抵抗感を下げる）
# **8. マルタと相性が良いポイント（任意で後から）**
- つなぎ役の手数料：1成立＝1マルタ（とか）
- レンタル保証的に：デポジット（マルタ預かり）も将来的に可能
- 「おすすめ本共有」は、\*\*“借りる＝返す”\*\*が自然に回るので相性良い
# **9. チャプ会で決めたいこと（議題）**
- 箱（Notion）を誰が作るか / どこに置くか
- 最初の“つなぎ役”を誰がやるか（1〜2週間だけでも）
- 受け渡しの基本ルール（ドーム受け渡し推奨で良いか）
- 初期登録ノルマ（各3件）をやるか
# **10. 次アクション（明日できるレベル）**
- Notionに「モノ一覧」「欲しいもの一覧」だけ作る（ログは後）
- チャプ会メンバーにテンプレ送って、各3件ずつ入力してもらう
- 入力された瞬間に、つなぎ役がマッチ候補を拾って声かけ開始
