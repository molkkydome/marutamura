# **0. ひとことで**
ヒトハブ＝「人のストーリー（価値観・思考変化）を資産化し、つながりを立体化する仕組み」
その結果として、相談・依頼・誘い・協働が自然に生まれる。
# **1. ヒトハブとは**
**定義**
ヒトハブは、個人の 価値観・思考変化・人生のターニングポイント を
チーム（チャプチェ会）が 深掘り → 言語化 → 整理 → 必要に応じ共有 する企画／仕組み。
**本質**
- 人を理解する
- 思考の流れを可視化する
- ストーリーを資産として残す
- コミュニティのつながりを立体化する
**前提**
単なるインタビュー企画ではなく、
コミュニティ文化と関係性の基盤づくり に寄与する取り組み。
# **2. 2つのレイヤー設計（ここが“完全版”の肝）**
**レイヤーA：ヒトハブ・カード（横のつながり即効）**
「この人ってどんな人？」が一瞬で分かる 情報箱。
相談・依頼・誘いの“取っかかり”をつくる。
例）
- 同じアーティスト好き → チケット当選で誘える
- 仕事カテゴリが分かる → お金払って相談・依頼できる
- 小1の壁・就職相談 → 話せる相手が見つかる
- なんとなく仲良くしたい → 声をかける理由ができる
**レイヤーB：ヒトハブ・ストーリー（縦の理解・文化資産）**
深掘りから得た 思考変化プロセス／価値観の源流 を構造化し、
本人にもコミュニティにも効く“理解のアーカイブ”にする。
Aは「入口」
Bは「深層」
Aだけでも回るし、Bがあるとコミュニティが別次元に強くなる。
# **3. 背景**
**チャプチェ会での経験（過去2ヶ月の確信）**
サノ・中さん・ししょーの深掘りを実施した結果、以下が明確になった。
- 思考が変化するプロセスに共通項がある
- 言語化が本人に巨大な内省を生む
- 他者理解が深まり、関係が立体的になる
- 承認・共感が自然に生まれ、コミュニティの質が上がる
- 深掘り → 言語化 → 共有は“継続して回す価値が高い”
**共通項（前提として共有済み）**
- 最低ラインの認識
- マルタの価値観の変化
- 承認の構造
- 今を生きる感覚への変化
- 役割確立 → 仲間 → 充実の循環<br>※基礎知として共有済みのため詳細省略。
**構想誕生の理由**
このプロセスが確実に「価値」になると確信し、
仕組みとして継続 させるためにヒトハブを計画した。
# **4. 目的（Purpose）**
**4-1. 個人の内省を最大化する（本人に返る価値）**
- 人生ストーリーが構造化され腑に落ちる
- 見えていなかった価値観が浮き彫りになる
- 思考の流れが整理され“安心感”につながる
- 行動の理由が明確になり自己肯定感が上がる
- 今後の意思決定がしやすくなる
**4-2. コミュニティの理解を深める**
- 価値観・背景・役割がわかる
- 誤解や摩擦が減る
- つながるべき人同士が自然に繋がる
- “濃いコミュニティ”が形成される
**4-3. 文化形成**
- 人を深掘りする文化
- 思考の変化を祝う文化
- ストーリーを残す文化
**4-4. 未来の価値創造**
- 人と人をつなぐハブ
- アーカイブ資産
- コミュニティ通貨との連動
- サブスク化
- 外部サービス化の可能性
# **5. ヒトハブに入れる情報（データ設計）**
**5-1. ヒトハブ・カード（軽量プロフィール）**
目的：取っかかりの最大化
- 呼び名（ニックネーム）
- 出没頻度／よく出る大会
- 住んでるエリア（市区町村レベル）
- 家族構成（ざっくりでOK）
- 仕事カテゴリ（ざっくりでOK）
- 趣味（モルック以外も）
- 好きなアーティスト／推し
- “相談OK”テーマ（例：転職、子育て、投資、家づくり等）
- “頼めること／得意なこと”（例：デザイン、動画、経理、段取り等）
- ひとこと（どんな人と繋がりたいか）
**5-2. ヒトハブ・ストーリー（深掘り成果物）**
目的：理解と文化資産
- ターニングポイント（出来事）
- 思考や行動が変わった理由
- 価値観の源流（どこで形成されたか）
- 安心のライン（最低ライン）
- 承認経験（何で満たされるか／何が刺さるか）
- コミュニティでの役割と立ち位置
- 行動原理／意思決定の軸
- 図解（思考変化マップ、価値観ツリー等）
# **6. プロセス（実際の流れ）**
**6-1. 深掘りインタビュー**
- 時間：60〜120分
- 対象：1名
- 方法：対面 or オンライン
- 進行：チャプチェ会メンバー複数名
- 録画：希望があれば
主なテーマ例
- ターニングポイント
- 思考や行動が変わった理由
- 安心のライン
- 価値観の源流
- 承認経験
- コミュニティでの役割と立ち位置
**6-2. 整理・文章化**
- 時系列整理、構造化
- 思考プロセスのストーリー化
- 価値観・行動原理の紐づけ
- 図解による可視化
- 本人が自分自身を深く理解できる形に整える
**6-3. 本人への返却・確認**
- 完成版送付
- 追加・修正反映
- 公開／非公開は本人が決定
**6-4. 共有（任意）**
- 本人希望の場合のみコミュニティ共有
- 人物理解が深まり関係が滑らかになる
# **7. 公開範囲・プライバシー設計（運用の要）**
**公開レイヤー（選べる）**
- 公開：全体向け
- メンバー限定：川西モルックの会／ドーム関係者のみ等
- 承認制：見せたい人だけ
- 非公開：本人返却のみ
**課金閲覧（アイデア）**
- 「みんなに知られたら嫌」層向けに<br>限定公開＋マルタ課金で閲覧 という選択肢もあり得る
- 実装する場合は、営業DM乱発・誤用を防ぐルールが必須
# **8. 得られる価値（短期・中期・長期）**
**短期：コミュニティの質が上がる**
- 背景理解の促進
- 協働のしやすさ
- 信頼関係の強化
**中期：個人の未来に効く**
- 自己理解の深化
- 思考変化が見え、安心と自信が増す
- 将来の判断基準が明確になる
**長期：コミュニティ資産の蓄積**
- 人物理解アーカイブ
- 文化の保全
- 新規メンバーの理解加速
# **9. 将来ビジョン（全可能性）**
- 人と人をつなぐハブ機能：深掘りデータが関係の地図になる
- 人生ストーリー図書館化（HITO LIBRARY）：人で文化を読める
- サブスク型プラットフォーム：許可制で読み放題アーカイブ
- 外部サービス化：深掘り＋文章化を対外有料提供
- コミュニティ通貨（マルタ／ハッピー）連動：応援・報酬文脈が明確に
- 運営の高度化：プロジェクト人選／適材適所／未来像設計へ活用
- 文化の創造：「人の変化っておもしろい」を当たり前にする
# **10. シンプルスタート（まず回す形）**
**スタート案（最小）**
1. ヒトハブ・カード（フォーム）を先に回す

4. 改善しながら人数を増やす
# **11. 次のアクション（チャプ会で決める）**
- 方向性の最終確認（チャプチェ会）
- 初回対象者の選定（川西モルックの会など）
- 深掘りインタビュー実施
- 文章化・可視化
- 本人へ返却・公開判断
- 改善点の整理、アーカイブ化検討へ
# **12. チャプ会での論点（決めどころ）**
- ヒトハブは「カード先行」か「ストーリー先行」か（結論はどっちでもよいが順番は決める）
- 公開範囲の基本方針（メンバー限定をデフォにする？）
- 運用責任者（承認・修正・削除・問い合わせ窓口）
- “課金閲覧”は今やるか、将来のオプションにするか
- 成功指標（登録数／事例数／協働の発生など）
